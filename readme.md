# Архитектура
## Движок
Основной цикл - в `Engine`, такт движка - `Engine::tick()`. В этом такте эвенты ввода передаются в подсистему ввода `InputSystem`, а так же вызывается такт графического движка `Renderer::tick()`. Система ввода и графический движок - синглтоны, т.е. существуют в единственном экземпляре на всю программу, создаются в конструкторе движка. 

## Графика
За один такт Рендерер проходится циклом по всем существующим `GameObject`, смотрит, есть ли у них текстуры, модели, собственные шейдеры (например отдельный шейдер для `Light`, для остальных стандартный), дёргает OpenGL функционал для смены контекста и отрисовки всех моделей. 

В Рендерере существует объект `Camera`. Он наследуется от `Translate`, что добавляет ему трехмерные координаты и методы. Позицию камеры и объектов рендерер передает шейдерам для просчёта матриц камеры.
